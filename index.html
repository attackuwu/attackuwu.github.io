Stochastic Kernel Docs

>> KERNEL ARCHITECTURE v1.0-alpha

The Stochastic Kernel is a hybrid microkernel architecture designed for high-throughput, low-latency operations in distributed systems. It utilizes a novel "stochastic scheduling" algorithm that predicts process CPU bursts based on historical heuristics.


[0x01] MEMORY MANAGEMENT UNIT (MMU) Implementation details

The MMU handles virtual-to-physical address translation using a 4-level page table structure (PML4). It supports 2MB and 1GB huge pages for performance optimization in database workloads.

// Paging Structure Definition

} __attribute__((packed));

The kernel implements a custom slab allocator for efficient kernel object management, reducing fragmentation and improving cache locality.


[0x02] PROCESS SCHEDULER (The "Stochastic" Engine)

Unlike CFM (Completely Fair Scheduler), the Stochastic scheduler uses a probabilistic model. Each task is assigned a probability distribution (Gaussian) representing its expected execution time.

  • - Priority Levels: Real-time (0-99), Normal (100-139).
  • - Time Slice: Dynamic, calculated via T = base * (1 + confidence_score).
  • - Context Switch Overhead: < 50ns on modern AVX-512 hardware.

[0x03] INTER-PROCESS COMMUNICATION (IPC)

IPC is achieved through zero-copy message passing using shared memory ring buffers. This eliminates the overhead of copying data between user space and kernel space.


>> CHANGELOG / VERSION HISTORY

v1.0-alpha (Current)

  • Initial release of the microkernel core.
  • Implemented basic VFS with RAM disk support.
  • Added VGA text mode driver for early boot logging.
  • Integrated basic shell (sh-compat).
  • Fixed critical race condition in spinlock acquisition.

v0.9-beta

  • Experimental ACPI support.
  • Added MultiProcessor Specification (MPS) parsing.
  • Refactored APIC interrupt controller driver.
  • Implemented basic TCP/IP stack (ARP, ICMP only).
  • Added "Stochastic" branding to bootloader.

v0.8-internal

  • Bootloader rewritten in Assembly (NASM).
  • transitioned from GRUB to custom Limine-based protocols.
  • Debugging symbol support added.
  • Kernel panic handler improved with stack tracing.
  • Initial support for NVMe drives (read-only).
  • Memory leak in heap allocator fixed.
  • Added `kprintf` function for formatted output.
  • Implemented GDT and IDT setup.
  • Added support for SSE/AVX instructions in kernel mode.
  • Fixed page fault handler infinite loop.

v0.7.5-nightly

Hotfix release for CI pipeline failures.

v0.7-legacy

Rewrote the scheduler in Rust (experimental branch), then reverted to C++.
